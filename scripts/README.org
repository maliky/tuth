# SQL Scripts Guide

This folder contains operational SQL scripts for merge prechecks, data repair,
and one-time backfills.

* How to run scripts

** Dockerized preprod

Run a script with:

#+BEGIN_SRC bash
cat scripts/<script_name>.sql | docker-compose -f docker-compose-preprod.yml exec -T db psql -U "$POSTGRES_USER" -d "$POSTGRES_DB"
#+END_SRC

For scripts that need parameters:

#+BEGIN_SRC bash
cat scripts/<script_name>.sql | docker-compose -f docker-compose-preprod.yml exec -T db psql -U "$POSTGRES_USER" -d "$POSTGRES_DB" -v source_id=1 -v target_id=2
#+END_SRC

** Dev / local

With direct psql:

#+BEGIN_SRC bash
psql -h "$POSTGRES_HOST" -p "$POSTGRES_PORT" -U "$POSTGRES_USER" -d "$POSTGRES_DB"
#+END_SRC

With Django dbshell:

#+BEGIN_SRC bash
python manage.py dbshell
#+END_SRC

Then run:

#+BEGIN_SRC sql
\i scripts/<script_name>.sql
#+END_SRC

* Script catalog

*** =backfill_curriculum_student_enrollment.sql=
Short description:
- Backfills CurriculumStudentEnrollment data from both:
  - =people_student.curriculum_id=
  - curricula inferred from student grades (section -> curriculum course).
- Writes to =people_curriculumstudentenrollment=.
- Keeps =Student.curriculum= authoritative for primary enrollment.
- Uses earliest graded semester as enrollment entry when inferred from grades.
- Safe to re-run (idempotent).

Example:

#+BEGIN_SRC bash
cat scripts/backfill_curriculum_student_enrollment.sql | docker-compose -f docker-compose-preprod.yml exec -T db psql -U "$POSTGRES_USER" -d "$POSTGRES_DB"
#+END_SRC
